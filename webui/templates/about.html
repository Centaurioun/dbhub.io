About Us

What is 3DHub.io

A collaboration platform for people to share and improve on 3D models, with a workflow similar to GitHub, Gitlab, and Gitea.

Why?

Justin Clift, the primary author for 3DHub.io (so far) has been doing 3D modelling for several years using both Open Source and commercial software. In that time, although various public websites have appeared which allow for model sharing (eg GrabCAD), none of them allow for good collaboration on models by multiple authors.

3DHub.io aims to change that, by providing a good platform where people can upload, download, modify, and contribute back changes to 3D models and assemblies. The entire code base is Open Source, so anyone wanting to run their own instance of it (eg locally), or extend and improve upon is is able to do so.

If we can generate sufficient ongoing revenue to make this all work, then yay, everyone wins! :)

How much is Open Source?

Everything is open source (AGPL3 or later).

Nothing held back, no "open core", etc. This is real, actual, proper, Open Source. Not the fake variety. :)

Features we intend to include

Most of these are still "in development" or will come along later, they're all on our definite To Do list:

Management of uploaded files. eg delete, rename, etc
Online viewer/editor, with access controls
Support for both individual models, and projects / assemblies containing many models
Teams + public/private projects
Versioning for models + basic "diff" support
An "Issues" section (trouble ticketing) for your projects
Forks, Pull Requests, Merging as per GitHub model
Branches, as per the git concept
Support for email replys to comments, for Issues/PR's/etc
Drag & drop image support for Issues/PR's/etc
Basic integration (send/list/receive) of 3D models from various Open Source CAD packages (TBD)
An API, so people can query/update their projects and models to develop their own integrations

How much will it cost?

Completely undetermined at this stage. ;)

The concept GitHub uses for pricing - free for public stuff, $ for private - is appealing, but may not work for 3D models. At least initially everything will be free, which should give us a start towards understanding usage patterns.

With that we can develop a workable model, though it may take a few iterations.
